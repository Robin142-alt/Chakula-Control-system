# Pitch Script

## 90-second version

"Chakula Control is a practical kitchen accountability system designed for schools where operations are real, messy, and often offline. Instead of blocking staff with strict validation, it saves data first, preserves raw notes, and flags risks later. The Storekeeper can issue stock quickly, the Cook can log leftovers in one or two taps, and the Principal sees only today's cost, cost per student, and the top three alerts. Behind the scenes, the system compares what was used against what should have been used, estimates waste, and highlights duplicate issues, missing leftovers, stock mismatches, and abnormal consumption. It is built to work on shared low-end devices and keep going even when the internet fails." 

## 3-minute version

"This project solves a common school problem: food leaves the store, meals are cooked, but no one can confidently explain daily food cost, waste, or missing stock. In many schools the process is partly on paper, entries are late, and internet access is unreliable.

So we designed the system around reality, not perfection. Every input screen is tolerant. We never reject entries. We preserve structured quantities and raw notes like '2 debes beans'. The browser stores data in IndexedDB first, then syncs later. That means the kitchen can continue operating even when the network is down.

The user experience is intentionally role-shaped. The Storekeeper gets fast stock issue and stock count screens. The Cook gets a very lightweight leftovers flow. The Accountant can review trends and reports. The Principal sees only the essentials: today's cost, cost per student, and up to three high-severity alerts.

For intelligence, the system answers four questions: what was used, what should have been used, what is missing, and whether the likely cause is waste, error, or possible theft. We do that with simple logic that still works when data is incomplete. In the demo week, the system catches duplicate beans issue entries, meals with no leftover logs, a high-consumption day, a low-consumption day, and stock mismatches in beans and oil.

The result is a system that is simple enough for daily kitchen use, but strong enough to give leadership meaningful cost visibility and accountability." 

## Closing line

"This is not a perfect-data system. It is a keep-the-kitchen-moving system that still produces useful accountability."
